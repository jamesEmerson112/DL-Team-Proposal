## Proposal Checklist

### Required Sections

- [ ] **Team Name**
- [ ] **Project Title**
- [ ] **Project Summary** (4-5+ sentences)
  - Problem statement
  - Background / motivation (why solve it? why is it interesting?)
  - Be specific — don't just say "I'll be working on X"
- [ ] **Approach** (4-5+ sentences)
  - What you will implement
  - What existing code you will use
  - Experiments you plan to try
  - Note: just downloading code and running it on a dataset is NOT sufficient — need thorough implementation, analysis, or theory
- [ ] **Resources / Related Work & Papers** (4-5+ sentences)
  - State of the art for the problem
  - Fine to implement existing approaches — just show you've researched what exists
- [ ] **Datasets** (include link)
  - Do NOT create/annotate your own data — it will consume the whole project
- [ ] **Group Members** (list all)

### Other Requirements

- [ ] Group size: 2-4 members
- [ ] Address ethical implications (if any negative societal/ethical concerns, list them)
- [ ] Ensure the project involves real implementation/analysis, not just running existing code

## Process

- [ ] Create group
- [ ] Draft proposal with group-mates
- [ ] Post draft on Ed if looking for teammates
- [ ] Finalize and submit by due date
- [ ] Verify all members included in Gradescope submission
